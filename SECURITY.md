# Security Policy

We take the security of AssetFlow CLI seriously. This document outlines how to report vulnerabilities, which versions are supported, and our policy on inspection and reverse engineering.

## Supported Versions

Only the latest major version of AssetFlow CLI receives active security updates.

| Version | Supported |
| :--- | :--- |
| >= 1.x.x |  Yes |
| < 1.0.0 |  No |

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public issue. Instead, report it privately via one of the following methods:

- **Email**: security@riish.in
- **GPG Key**: [Provide fingerprint if applicable, or link to keyserver]

Please include:
- A description of the vulnerability.
- Steps to reproduce (proof-of-concept script or commands).
- The impact it could have on users or workspaces.

We will acknowledge receipt of your report within 48 hours and provide a detailed response with a resolution timeline.

## Disclosure Policy

- Once a report is received, we will investigate and verify the vulnerability.
- A fix will be developed and tested.
- We follow coordinated disclosure. We request that you do not disclose the vulnerability publicly until a patch is released to protect users.
- A public security advisory will be published along with the patched release.

## Inspection & Reverse Engineering Policy

Inspection, reverse engineering, or analysis of the software does not grant any additional rights beyond those explicitly provided by the license under which it is distributed. Any intellectual property extracted through such methods remains the exclusive property of AssetFlow and cannot be used commercially, redistributed, or repackaged without a separate commercial license.
